# 통합단말 신문고 Streamlit 시연 완성본

OO은행 영업점 직원용 내부 VOC 시스템을 시연하기 위한 Streamlit 앱입니다. 첫 화면은 게시판이 아니라 가상의 통합단말 업무 화면이며, 업무 화면 안의 `통합단말 신문고` 버튼을 누르면 현재 화면 위에 모달 접수창이 열리고 화면 맥락이 자동 전달됩니다.

## 설치

```bash
python3 -m pip install -r shinmungo/requirements.txt
```

## 실행

```bash
streamlit run shinmungo/app.py
```

최초 실행 시 `./shinmungo/shinmungo.db`가 생성되고, `./data/terminal_synthetic_xlsx/`의 raw 엑셀 3개를 기준으로 시드 데이터가 들어갑니다.

## 아키텍처

```text
통합단말 화면
  -> session_state 화면 컨텍스트
  -> 신문고 모달 접수
      -> 민감정보 경고
      -> 화면 영역 선택 컨텍스트
      -> 임베딩 유사검색
      -> 도움됨 종결/신규접수
      -> 자동분류 + 담당부서 라우팅
      -> Ticket / History / Follower
  -> 관리자
      -> 유사 티켓
      -> RAG 답변 초안
      -> Answer 저장 + 상태 변경 + Notification
  -> 내 접수내역
      -> 진행상황
      -> 유사 사례 처리 결과 요약
      -> 팔로우 알림
  -> 대시보드
      -> SQL/pandas 집계
      -> 중복 클러스터링
      -> 우선순위 계산
```

## 시연 시나리오

1. 앱 실행 후 `통합단말 화면`에서 `111 계좌 + 카드 동시신규` 또는 `827 퇴직연금 보유상품변경`을 선택합니다.
2. 업무 화면 우측 상단의 `통합단말 신문고` 버튼을 클릭해 모달 접수창을 엽니다.
3. 모달 안에서 `기능 지정하기`를 눌러 팝업을 잠시 닫고, 통합단말 화면에서 수정 요청할 업무 영역을 클릭합니다.
4. 팝업이 다시 열리면 선택 영역 칩과 화면번호, 화면명, 업무영역이 자동 표시된 상태를 확인하고 제목/상세 내용을 작성합니다.
5. `접수하기`를 누르면 시스템이 먼저 유사 요청을 확인합니다.
6. 유사 요청이 있으면 대표 카드 3개를 확인하고, 카드별 `이 내용이 도움이 되었어요`로 신규 접수를 중단하거나 팝업 하단의 `새로운 요청으로 접수해주세요`로 새로 접수합니다.
7. 접수 완료 메시지에서 접수번호, 배정 부서, 자동 분류를 확인하고 `닫기`로 업무 화면에 복귀합니다.
8. 사이드바 역할을 `관리자`로 바꾼 뒤 `관리자` 페이지에서 선택 영역 상세, 답변 초안을 확인하고 편집해 저장합니다.
9. 관리자 화면에서 상태 변경 또는 AI 분류 보정을 수행합니다.
10. 역할을 `직원`으로 되돌리고 `내 접수내역`에서 진행상황, 유사 사례 요약, 팔로우 중인 요청, 알림을 확인합니다.
11. `대시보드`에서 중복 클러스터, 우선순위, 분류/부서/상태 분포를 확인합니다.

`통합단말 신문고` 접수는 업무 화면 위의 모달 팝업에서만 진행합니다. 사이드바에는 별도 신문고 접수 페이지를 노출하지 않습니다.

## 데이터와 범위

- 화면 기준: `111 계좌 + 카드 동시신규`, `827 퇴직연금 보유상품변경`
- 담당부서: `111=금융서비스개발부`, `827=퇴직연금사업부`
- raw 요청 240건과 답변/처리이력 328건을 SQLite에 보존합니다.
- AI 시연용 유사쌍 요청 30건을 추가로 생성합니다.
- 현행 raw 데이터에 없는 별도 문답 메뉴는 사용하지 않고, `기존 답변 사례`와 `기존 유사 문의` 표현만 사용합니다.
- 유사 검색은 `sentence-transformers`의 `jhgan/ko-sroberta-multitask`를 기본으로 사용합니다.
- 로컬 임베딩 모델 로딩에 실패하면 앱 부팅을 막지 않고 `간이 로컬 벡터`로 폴백합니다.
- 자동 분류는 프로젝트 루트 `.env`의 `OPENAI_API_KEY`가 있으면 `gpt-5.5`를 사용하고, 실패하거나 키가 없으면 키워드 규칙으로 폴백합니다.
- RAG 답변 초안과 유사 사례 요약도 `OPENAI_API_KEY` 기반 `gpt-5.5`를 사용하며, 키가 없거나 호출 실패 시 템플릿 기반 오프라인 폴백으로 동작합니다.

## AI 기능

- 접수 전 유사 검색: `screen_code + title + content`를 임베딩하고 같은 화면번호 요청에는 유사도 가중치를 부여합니다.
- 영역 선택: 신문고 팝업의 `기능 지정하기`로 통합단말 업무 화면의 지정 가능 영역을 선택하고 `selected_area_json`으로 저장합니다.
- 중복 차단 UX: `접수하기` 시 유사 요청을 자동 확인하고, 대표 카드에서 기존 요청 팔로우 또는 신규 접수를 선택할 수 있습니다.
- 자동 분류: `버그`, `UI불편`, `기능개선`, `성능`, `업무절차혼선`, `기타`, `미분류` 중 하나로 저장합니다.
- 관리자 보정: 관리자가 AI 분류를 수정하면 `ClassificationFeedback`에 보정 이력이 남습니다.
- RAG 답변 보조: 관리자 화면에서 유사 답변과 처리 이력을 근거로 답변 초안을 생성하고 `Answer`로 저장합니다.
- 요청자 요약: 내 접수 상세에서 유사 사례 처리 결과 요약을 확인할 수 있습니다.
- 중복 클러스터링: 임베딩 유사도 기반 연결요소로 중복 요청 군집을 계산합니다.
- 우선순위: 군집 크기, 팔로워 수, 분류 영향도, 미해결 경과일을 기준으로 점수를 계산합니다.
- 관리자 대시보드: 총 접수, 미처리, 지연, 완료율, 화면/상태/분류/부서 분포, 화면별 중복 클러스터 버블 차트와 카드 상세를 제공합니다.
- 알림 mock: 팔로우 중인 요청의 상태가 변경되면 `Notification` 테이블에 알림이 기록됩니다.
- 민감정보 경고: 주민등록번호와 계좌번호로 보이는 숫자 패턴을 접수 전 경고합니다.

## 환경변수와 모드

- `.env`에 `OPENAI_API_KEY`가 있으면 자동분류와 RAG 초안/요약에서 `gpt-5.5`를 사용합니다.
- 키가 없거나 API 호출이 실패하면 규칙 기반 분류와 템플릿 기반 RAG 폴백으로 동일한 시연 흐름을 유지합니다.
